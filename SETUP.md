# Setup Guide

## 1. Create the profile repository

Create a new public GitHub repository named exactly:

```text
oomnii
```

GitHub displays the root `README.md` from this repository at the top of your profile.

## 2. Upload these files

Upload the complete contents of this folder so the repository looks like:

```text
oomnii/
├── README.md
└── assets/
    ├── header.svg
    └── Om-Seth-Resume.pdf
```

Do not upload the outer `oomnii-github-profile` folder itself.

## 3. Update your GitHub public profile

Recommended bio:

```text
Software Engineer | AI/ML Enthusiast | Building full-stack products, NLP systems, and interactive web experiences
```

Recommended website:

```text
https://om-seth-portfolio.vercel.app/
```

Recommended social links:

- LinkedIn: https://www.linkedin.com/in/om-seth-94ab523b9/
- LeetCode: https://leetcode.com/u/oomniiii/
- Codeforces: https://codeforces.com/profile/omniiii

Use a professional photo, a clean illustrated portrait, or a simple OS monogram instead of the default generated avatar.

## 4. Recommended pinned repositories

Pin these in this order:

1. Portfolio-Codebase
2. Smart-Hiring-and-Resume-Scorer
3. Energy-Efficient-Node-Scheduling-with-AI-Driven-Optimization
4. ABHYAAS-OnlineExam-Portal
5. Homelab-Vault-Secure-Manager
6. HTTP-vs-WebSocket or LeetCode-oomniiii

Unpin small classroom exercises such as the basic calculator and web-engineering lab.

## 5. Improve each pinned repository

For each pinned project, add:

- A strong one-sentence description
- A screenshot or demo GIF
- A live deployment link where available
- Feature list
- Architecture explanation
- Setup commands
- Tech stack
- License
- Repository topics
- A 1280 × 640 social-preview image

## 6. Dynamic-card reliability

The README uses public GitHub statistics services. These services can occasionally be rate-limited or unavailable. Once the design is approved, self-host the statistics cards or generate them through GitHub Actions for maximum reliability.
